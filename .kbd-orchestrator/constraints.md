Generated by `/kbd-init` on 2026-06-23.

## Blocking Constraints

- Never commit secrets, JWT signing keys, or production database credentials to version control.
- Never expose the admin server (port 4457) to the public internet.
- Never break existing unit tests without updating or replacing them.
- Never change configuration priority order (CLI > env > YAML) without updating tests and docs.

## Warning Constraints

- Prefer `anyhow` for application error paths and `thiserror` for library error types.
- Avoid `unwrap`/`expect` outside of tests and initialization that cannot fail.
- Keep the proxy port (4456) and admin port (4457) concerns separated.
- Maintain hot-reload behavior when modifying configuration loading code.
- Follow existing module structure under `src/` when adding new features.
